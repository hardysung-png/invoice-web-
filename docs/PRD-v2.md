# 노션 기반 견적서 관리 시스템 v2 PRD

## 🎯 핵심 정보

**목적**: v1(노션 CMS 견적서 생성 + 이메일/텔레그램/링크 공유 + 어드민 대시보드)을 기반으로, **견적서 상태 추적**, **Slack 실시간 알림**, **수신자 수락/거절**, **네고 트리** 기능을 추가하여 거래 템포를 가속하고 협상 히스토리를 추적 가능하게 한다.

**v1 연속성 (Non-Negotiable)**:

- 노션 CMS를 데이터베이스로 유지 (신규 DB 도입 X)
- Next.js 15.5.3 App Router, Turbopack, shadcn/ui 그대로
- 이메일/텔레그램 발송 채널 유지
- 기존 견적서 고유 URL(`/invoice/[notionPageId]`) 구조 유지

**v2가 변경하지 않는 것**:

- v1에서 구축된 어드민 인증/세션 흐름
- PDF 생성 파이프라인
- 기존 Notion Invoice/Items 스키마의 필드 (신규 필드만 추가)

## 🚶 사용자 여정 (v2 확장)

### 견적서 작성자 (관리자)

```
1. 노션에 견적서 작성 (v1 그대로)
   ↓
2. 상태: "발송됨" 자동 설정, 발송일 +7일 = 만료일 계산
   ↓
3. Slack 전용 채널에 "견적서 발송" 알림 수신
   ↓
4. 수신자 조회 시 Slack 알림 ("검토중"으로 상태 전이)
   ↓
5-A. 수락/거절 시 Slack 알림 → 종료
5-B. 네고 제출 시 Slack 알림 → 어드민 화면에서 수락 또는 역제안
     → 역제안 시 자식 견적서 자동 발행
```

### 클라이언트 (수신자)

```
1. 링크 접속 (v1 그대로)
   ↓
2. 최신 자식 견적서로 자동 리다이렉트 (네고 진행 중인 경우)
   ↓
3. 가격 UI: 원본가(취소선) + 현재가, 자세히 보기 아이콘
   ↓
4-A. [수락] 버튼 → 완료
4-B. [거절] 버튼 → 사유 입력 → 완료
4-C. [네고] 제출 → 항목별 수량·가격 수정 + 메모 (floor_price 이하 차단)
     → 어드민 대기
```

만료(발송일 +7일 이후) 시점에는 4의 모든 액션이 잠기며, "만료된 견적서입니다. 재발행을 원하시면 담당자에게 문의해 주세요" 안내 페이지로 대체된다.

## ⚡ 기능 명세

### F101 견적서 상태 필드 확장

| 항목                 | 내용                                                                                                                                                                                              |
| -------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **상태값**           | `발송됨`, `검토중`, `수락`, `거절`, `네고중`, `만료`                                                                                                                                              |
| **상태 전이**        | 발송됨 → 검토중(수신자 첫 조회) → 수락/거절/네고중. 만료는 발송일 +7일 경과 시 어떤 상태에서도 진입 가능                                                                                          |
| **만료 로직**        | 발송일 +7일 **고정**. 연장 없음. 만료 후 링크 접속 시 안내 페이지만 노출                                                                                                                          |
| **추가 Notion 필드** | `floor_price` (항목별 최소 제안가, Number), `max_nego_rounds` (Number, 기본 3, 견적서별 오버라이드 가능), `expires_at` (Date, 발송일 자동 +7일), `parent_invoice` (Relation, 자식 견적서용)       |
| **타입 확장**        | `src/types/invoice.ts`의 `InvoiceStatus`를 `'sent' \| 'reviewing' \| 'accepted' \| 'rejected' \| 'negotiating' \| 'expired'`로 확장. 기존 `pending/approved/rejected` 값은 마이그레이션 매핑 처리 |

### F102 Slack 알림 파이프라인

| 항목                 | 내용                                                                                                                               |
| -------------------- | ---------------------------------------------------------------------------------------------------------------------------------- |
| **채널**             | **전용 채널 신설** (기존 채널 재활용 금지)                                                                                         |
| **메시지 형식**      | 이벤트별 **개별 메시지** (스레드 누적 X). 타이틀 + 견적서 번호 + 클라이언트명 + 링크 + 컨텍스트 필드                               |
| **트리거 전체 수신** | 발송 / 조회(발송됨→검토중) / 수락 / 거절(+사유) / 네고 제출(+제안) / 만료 임박(6일 차, "내일 만료") / 만료됨                       |
| **구현 방향**        | Slack Incoming Webhook URL 환경변수(`SLACK_WEBHOOK_URL`)로 주입. 기존 해브어 노션 프로모션 자동화 파이프라인 재활용 가능 여부 검토 |

### F103 수락/거절 버튼

| 항목          | 내용                                                                                 |
| ------------- | ------------------------------------------------------------------------------------ |
| **수신자 UI** | 견적서 하단 `수락` / `거절` 버튼. 거절 선택 시 **사유 입력창 필수**                  |
| **어드민 UI** | 대시보드 목록에서 견적서 상태 실시간 표시. 상태 전이 로그 확인 가능                  |
| **서버 액션** | Next.js Server Action으로 Notion 상태 업데이트 → Slack 알림 → 수신자 화면 리다이렉트 |

### F104 네고 기능 (트리 구조)

| 항목                 | 내용                                                                                                                                   |
| -------------------- | -------------------------------------------------------------------------------------------------------------------------------------- |
| **구조**             | 원본 견적서 상태: `네고중` 유지. 역제안 시 **자식 견적서 신규 페이지 생성**, `parent_invoice`로 연결. 협상 히스토리 전체가 트리로 추적 |
| **링크 동작**        | 기존 링크 유지. 수신자 접속 시 **최신 자식 견적서로 자동 리다이렉트**                                                                  |
| **회차 제한**        | 기본 **3회**, 견적서별 `max_nego_rounds` 오버라이드 가능. 초과 시 네고 버튼 비활성화                                                   |
| **수신자 제안 가드** | `floor_price`를 수신자에게 **명시적으로 공개**. 최소가 이하 제출은 서버 액션에서 차단 + 인라인 에러                                    |
| **제출 항목**        | 항목별 수량 변경 / 항목별 가격 제안 / 자유 메모                                                                                        |
| **가격 UI (공통)**   | 원본가(취소선) + 현재가. **아이콘 버튼 클릭** 시 전체 제안 로그 모달 (마우스 오버 금지, 모바일 호환)                                   |
| **로그 표기**        | `1차: 50,000원 [우리 제안]` / `2차: 45,000원 [고객 제안]` / `3차: 42,000원 [우리 제안]` 형식. 주체 라벨 명시                           |
| **어드민 네고 처리** | 제안 상세 확인 후 `수락` 또는 `역제안` 선택. 역제안 클릭 시 자식 견적서 생성 폼 오픈 → 발행 시 Slack 알림                              |

### F105 카카오톡 공유 (v2 범위 외)

구현하지 않음. 재개 시 다음 확인 필요:

## ✅ v2 성공 기준

1. 6개 상태가 Notion `Select`에 등록되고 어드민 대시보드에 정확히 렌더링됨
2. 수신자 링크 첫 조회 시 `발송됨` → `검토중` 자동 전이 + Slack 알림 1건
3. 수락/거절 시 Notion 상태 업데이트 + Slack 알림 + 안내 화면 전환
4. `floor_price` 이하 네고 제출이 UI와 서버 양쪽에서 모두 차단됨
5. 역제안 시 자식 견적서가 생성되고, 기존 링크 접속이 최신 자식으로 리다이렉트됨
6. 네고 로그 모달이 버튼 클릭으로 열리고 모바일에서 정상 동작
7. 만료 크론이 `expires_at` 도달 견적서를 `expired`로 전환, 6일 차에 "내일 만료" Slack 알림
8. 만료된 견적서 링크 접속 시 드라이한 안내 페이지 노출, 모든 액션 버튼 비활성화
9. v1 기능(PDF 다운로드, 링크 복사, 어드민 목록) 전부 회귀 없음
